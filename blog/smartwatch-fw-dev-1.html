The need for more power Written on 17/04/16

Current issues

So far the project is progressing well, I have laid the ground works for notification reception on the smartwatch but I have few problems with the current setup:

  • RAM, the arduino nano only has 2KB of SRAM, storing large text notifications will quickly use all out precious RAM
  • Serial buffer issues, arduino missing data with software serial
  • Runs on 5 volts where all our other devices use 3.3v (Not a huge deal as we could switch to a 3.3 nano or pro mini)

The solution

A arduino compatible board with more RAM and multiple dedicated Serial ports was the solution. I went with the Teensy LC due to its great compatibility with arduino software and it's low cost to power ratio. This board packs a 32 bit Cortex M0+ running at 48Mhz. So I bought two boards, one for development and one for the final smartwatch, the switch over was very easy, downloading the teensyduino software and selecting teensy LC from the board list in the arduino IDE I had it up and running like on the arduino nano.

Once I got it running I used one of the 3 hardware serial ports and wired up my HC-06. I opened up a bluetooth terminal app I got fromt eh playstore and started spamming long messages. It worked flawlessly, now onto the android app...
