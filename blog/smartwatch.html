Arduino Smart Watch Written on 26/03/16

Introduction

I am starting this blog after doing alot of the ground work for this arduino smart watch. In this first post I will discuss what I have got so far and how I got here.

Hardware choices

A quick overview of the main hardware in the prototyping phase.

  • Arduino Nano 328p
  • 1.3" OLED using the SSH106 driver chip
  • HC-06 Bluetooth module
  • 240mAh LiPo Battery 3.7v
  • 3 tactile switches for input

I chose the Arduino Nano, with the 328p achitecture version, A simple robust microcontroller with a lot of library support. The 1.3" OLED was the biggest oled I could find on ebay, but the driver chip ended up being an issue which I will discuss below. The HC-06 is a very common bluetooth module that can be picked up for a few pounds on ebay or amazon. Another common bluetooth module is the HC-05, the main difference between the two is that the HC-05 can switch roles between slave and master, unlike the HC-06 which is only availble as a slave.

Software

There are two sides to the software story, the software running on the watch (A.K.A firmware) and the software running on the android device. I will leave the android app development for another post as there is much to write about.

Designing the Arduino smart watch firmware

The firmware need to do a few things:

  • Recieve information from the app via serial
  • Give a menu for users to navigate the info sent to the watch
  • Display info on a OLED
  • Handle user input

The HC-06 provide a serial connection that can read by the arduino, so we just have to keep checking the serial buffer of the arduino for data from the phone.
